# FileAgent — полное описание проекта

## Что это

FileAgent — персональный AI-агент для управления документами и заметками. Работает на вашем сервере, подключается через Telegram, веб-панель, REST API и MCP (Claude Desktop/Code).

Вы отправляете документ — система автоматически распознаёт текст, определяет тип, извлекает данные, сохраняет в структурированное хранилище и делает всё это доступным для поиска. Вы надиктовываете голосовую заметку — система транскрибирует, категоризирует, извлекает метрики (калории, вес, настроение) и раскладывает по Obsidian-совместимому хранилищу.

## Обработка документов

### 13-шаговый конвейер

Каждый загруженный файл проходит через автоматический конвейер:

1. **Приём** — проверка размера, расширения, целостности
2. **Авто-кадрирование** — OpenCV детектирует документ на фоне и обрезает края (только для фото)
3. **Дедупликация** — SHA-256 проверка на точные дубликаты
4. **Парсинг** — извлечение текста: PyMuPDF (PDF), Tesseract OCR (изображения), python-docx (DOCX)
5. **Классификация** — LLM определяет категорию, тип документа, теги, приоритет
6. **Маршрутизация** — подбор YAML-скилла по категории/ключевым словам
7. **Извлечение полей** — LLM достаёт структурированные данные (имена, суммы, даты, диагнозы)
8. **Сохранение** — файл на диск с категоризацией (локально, S3 или Google Drive)
9. **Эмбеддинг** — мультимодальный Gemini Embedding 2 (768 измерений) + проверка семантических дубликатов
10. **Метаданные** — запись в SQLite с FTS5-индексацией
11. **Инвалидация кэша** — обновление поискового кэша
12. **Обновление инсайтов** — фоновый пересчёт аналитики по категории
13. **Авто-напоминания** — создание напоминаний по дедлайнам из документа

### Поддерживаемые форматы

PDF, PNG, JPG, JPEG, HEIC, DOCX, TXT, CSV, XLSX, MD — до 50 МБ на файл.

### Многостраничное сканирование

Команда `/scan` запускает сессию сканирования: отправляете фотографии страниц одну за другой, система собирает их в PDF. Или отправьте несколько фото разом — они автоматически склеятся. OpenCV обрезает фон с каждого кадра.

## Семантический поиск (RAG)

Все документы индексируются через мультимодальные эмбеддинги Gemini 2. При запросе:

1. Текст запроса эмбеддится как `RETRIEVAL_QUERY`
2. Qdrant находит ближайшие чанки по косинусной близости
3. LLM синтезирует развёрнутый ответ на основе найденных документов
4. Ответ включает конкретные данные: даты, суммы, имена, номера

Поиск работает из Telegram (`/search`), веб-панели, API и MCP. Поддерживает контекст диалога — можно задавать уточняющие вопросы.

## Аналитика документов

Команда `/analytics` запускает мультидокументный анализ. Например:

- "Проанализируй гемоглобин за последний год" → извлечёт значения из всех анализов, построит график с референсными диапазонами, опишет динамику
- "Сколько я потратил за квартал" → суммирует инвойсы, покажет тренд расходов

Генерирует PNG-графики через matplotlib и текстовые саммари.

## AI Insights

Система анализирует все документы в каждой категории и формирует:

- Обзор ключевых фактов и проблем
- Практические рекомендации (5-8 пунктов)
- Дедлайны и истекающие документы
- Веб-исследование через Tavily (опционально)

Ежедневные советы отправляются в Telegram в 9:00 и 20:00.

## Smart Notes — умные заметки

### Как это работает

1. Надиктовываете голосовое сообщение в Telegram (или пишете `/note текст`)
2. OpenAI Whisper транскрибирует голос в текст
3. **NoteAgent** (фоновый AI-агент) обрабатывает заметку:
   - Определяет категорию и подкатегорию
   - Извлекает структурированные данные (калории, вес, настроение и т.д.)
   - Генерирует заголовок, саммари, теги
   - Находит связанные заметки через семантический поиск
   - Создаёт Obsidian-совместимый .md файл с wikilinks
   - Эмбеддит в Qdrant для будущего поиска
4. Оригинал транскрипции сохраняется и в БД, и в .md файле

### Категории заметок (12)

| Категория | Подкатегории | Что извлекается |
|-----------|-------------|-----------------|
| **health** | symptoms, medications, doctor_visits, mental_health | симптом, тяжесть (1-10), локация боли, лекарство, диагноз |
| **food** | meals, water, supplements | тип приёма пищи, калории (поэлементно), продукты, вода |
| **fitness** | weight, exercise, sleep, tracker | вес (кг), тип упражнения, длительность, часы сна, качество сна (1-10) |
| **business** | meetings, calls, tasks, projects | участники, задачи с приоритетом и дедлайном |
| **personal** | mood, reflections, gratitude, relationships | настроение (1-10), эмоция |
| **finance** | expenses, income, investments | сумма, валюта, описание |
| **learning** | books, courses, insights | название, ключевой вывод |
| **goals** | short_term, long_term | описание цели, дедлайн |
| **people** | clients, contacts, family_members | имя, контекст встречи |
| **ideas** | — | краткое описание идеи |
| **family** | events, milestones | событие, дата |
| **auto** | tesla, tools, repairs | марка/модель, проблема, стоимость |

### AI-классификация

Каждая заметка анализируется LLM, который извлекает:

- **Категория + подкатегория** — из 12 категорий
- **Confidence** (0.0-1.0) — уверенность классификации. При confidence < 0.5 заметка попадает в `_inbox/` для ручного разбора
- **Sentiment** (-1.0 to +1.0) — тональность заметки (негативная → позитивная)
- **Energy** (1-10) — уровень энергии, если упоминается
- **Mood score** (1-10) — настроение, определяется по тону если не указано явно
- **Structured data** — специфичные для категории данные (калории, вес, задачи и т.д.)
- **Action items** — задачи с приоритетом (high/medium/low) и дедлайном
- **Related categories** — вторичные категории для cross-linking

Если заметка затрагивает несколько тем ("поел после встречи с клиентом, голова болит"), выбирается основная категория, остальные попадают в теги и связи.

### Obsidian Vault

Заметки раскладываются по папкам категорий:

```
~/ai-agent-files/notes/
  _inbox/                         ← неуверенные классификации (confidence < 0.5)
  _archive/                       ← оригиналы из inbox watcher
  daily/
    2026-03-28.md                 ← ежедневный обзор (Map of Content)
    week-2026-W13.md              ← недельный отчёт
  food/
    _moc.md                       ← индекс категории (Map of Content)
    2026-03-28-lunch.md
  health/
    _moc.md
    symptoms/...
  fitness/weight/...
  business/meetings/...
  people/clients/...
  ideas/...
  family/events/...
  auto/tesla/...
```

Каждый файл содержит YAML frontmatter (дата, категория, теги, note_id, confidence, sentiment, energy, mood), секцию данных, оригинальный текст и wikilinks к связанным заметкам.

Для каждой категории автоматически генерируется `_moc.md` — индексный файл с облаком тегов, списком подкатегорий и ссылками на все заметки.

### Трекинг метрик

Из заметок автоматически извлекаются числовые метрики и сохраняются для анализа трендов:

| Метрика | Единица | Источник |
|---------|---------|----------|
| Калории | kcal | Оценка по описанию еды (поэлементно) |
| Вес | кг | Упоминание в заметке |
| Настроение | 1-10 | Явная оценка или определение по тону |
| Энергия | 1-10 | Самооценка уровня энергии |
| Sentiment | -1.0 to +1.0 | Автоматический анализ тональности |
| Сон (часы) | ч | Упоминание или check-in |
| Качество сна | 1-10 | Самооценка |
| Вода | мл | Упоминание |
| Тренировки | мин | Упоминание |
| Шаги | шт | Упоминание или трекер |
| Расходы | USD/RUB/EUR | Финансовые заметки |
| Тяжесть симптома | 1-10 | Медицинские заметки |
| Стоимость авто | USD | Заметки про машины/ремонт |

### Вечерний Check-in

В 21:00 (настраивается) бот проверяет, какие категории заметок пропущены за день, и задаёт вопросы:

- "Что ты ел сегодня?" (если нет food-заметок)
- "Как настроение? Оцени от 1 до 10" (inline-кнопки)
- "Вес сегодня?"
- "Сколько спал прошлой ночью?"

Каждый ответ сохраняется как заметка и обрабатывается NoteAgent. В конце — итоговое сообщение с метриками дня.

### Аналитика заметок

Веб-дашборд `/notes` показывает:

- Карточка дня (заметки, настроение, энергия, калории, вес)
- Графики трендов: настроение, калории, вес, сон (14-30-90 дней)
- Распределение по категориям
- Корреляции ("В дни с 7+ часами сна настроение в среднем 7.8 vs 5.2")
- Поиск по заметкам (FTS5 + фильтры по категории)

### Недельный анализ

Каждое воскресенье в 22:00 система генерирует AI-отчёт за неделю:

- Сводная таблица: день за днём (калории, сон, настроение, энергия, заметки)
- **Паттерны**: "настроение падает по средам — самый загруженный рабочий день"
- **Корреляции**: "спишь лучше в дни с тренировками"
- **Тренды**: "калории на 15% ниже прошлой недели"
- **Алерты**: "пропустил обед 3 дня"
- **Стрики**: "5 дней подряд с заметками!"
- **Рекомендации**: конкретные, основанные на данных (не generic)

Отчёт сохраняется как Obsidian note (`daily/week-YYYY-Www.md`) и отправляется в Telegram.

Также доступен по команде `/analyze` в любой момент.

### Ежедневная сводка (23:55)

Перед полуночью система автоматически:
- Генерирует/обновляет daily MOC в Obsidian
- Отправляет итоги дня в Telegram (все метрики + стрик)

### File Watcher (inbox monitor)

Система мониторит папку `_inbox/` каждые 10 секунд. Если туда попадает новый .md файл (например, от FAG или другого транскрибера):
1. Читает содержимое
2. Сохраняет как заметку в БД
3. Обрабатывает через NoteAgent (категоризация, метрики, vault)
4. Перемещает оригинал в `_archive/`

Это позволяет интегрировать любые внешние источники заметок — достаточно положить .md файл в inbox.

## Система скиллов

Скиллы определяют, как обрабатывать каждый тип документа. Это YAML-файлы в папке `skills/`:

```yaml
name: health
display_name: "Медицина / Health"
category: health
routing_rules:
  keywords: [анализ, diagnosis, prescription, lab]
  patterns: ["\\d+.*mmol", "гемоглобин"]
  mime_types: [application/pdf, image/jpeg]
extraction:
  fields:
    - name: patient_name
      description: "Имя пациента"
    - name: key_findings
      description: "Основные результаты"
  custom_prompt: "Извлеки данные из медицинского документа..."
response_template: |
  🏥 {document_type}
  👤 {patient_name}
  📋 {key_findings}
```

Скиллы перезагружаются каждые 30 секунд — изменения подхватываются без перезапуска.

Встроенные скиллы: Health, Business, Receipts, Personal.

## REST API

9 эндпоинтов с Bearer-token аутентификацией. Два режима доступа: Lite (только чтение) и Full (полный).

```
GET  /api/v1/stats              — статистика архива
GET  /api/v1/files              — список файлов (пагинация)
GET  /api/v1/files/{id}         — метаданные файла
GET  /api/v1/files/{id}/text    — извлечённый текст
GET  /api/v1/files/{id}/download — скачать файл
GET  /api/v1/search?q=запрос    — семантический поиск
POST /api/v1/files/upload       — загрузить файл (base64)
DELETE /api/v1/files/{id}       — удалить файл
```

Rate limiting: поиск 30/мин, загрузка 10/мин, удаление 20/мин.

## Хранилища

### Файлы

Три бэкенда на выбор:

| Бэкенд | Описание |
|--------|----------|
| **Local** | Диск с категоризированными папками (по умолчанию) |
| **S3** | AWS S3, MinIO, Wasabi, Backblaze B2 |
| **Google Drive** | Папочная структура через Service Account |

Все бэкенды поддерживают шифрование файлов (AES-256-GCM).

### База данных

SQLite + aiosqlite (WAL mode) — 12+ таблиц:

- `files` — метаданные документов
- `notes` — заметки (15 колонок, включая категорию и метрики)
- `note_data` — временные ряды метрик (калории, вес, настроение...)
- `note_links` — связи между заметками
- `processing_log` — история выполнения конвейера
- `llm_usage` — расход LLM (токены, стоимость)
- `search_cache` — кэш поисковых запросов
- `reminders` — автоматические напоминания
- `insights` — AI-аналитика по категориям
- `folders` — пользовательские папки
- `api_keys` — Bearer-токены API
- `secrets` — зашифрованные API-ключи

FTS5 полнотекстовый поиск по файлам и заметкам.

### Векторная БД

Qdrant (remote) — коллекция `file_agent_v2`:
- 768-мерные эмбеддинги (Gemini Embedding 2)
- Косинусная метрика
- Мультимодальные точки (PDF, изображения, аудио)
- Текстовые чанки (400 слов, 50 перекрытие)
- Точки заметок (type="note" в payload)

## Безопасность

Подробное описание всех уровней защиты — в [SECURITY_OVERVIEW.md](SECURITY_OVERVIEW.md).

Краткое резюме:

- **AES-256-GCM** — шифрование файлов на диске и данных в БД
- **Argon2id** — защита мастер-пароля (64 МБ на попытку, ~2 попытки/сек)
- **2FA** — мастер-пароль + физический ключ (USB-флешка)
- **Telegram hardening** — привязка к owner_id, авто-удаление, PIN-код
- **Web security** — bcrypt, CSP, session protection, rate limiting
- **API control** — Bearer-токены с разграничением прав
- **Нулевая телеметрия** — всё на вашем сервере, ничего не отправляется

## Технический стек

| Компонент | Технология |
|-----------|-----------|
| Backend | Python 3.14, FastAPI, uvicorn |
| Telegram | python-telegram-bot 22.7 |
| LLM | litellm (Claude, GPT, Gemini) |
| Embedding | Gemini Embedding 2 (768-dim multimodal) |
| Vector DB | Qdrant |
| Metadata DB | SQLite + aiosqlite (WAL, FTS5) |
| Parsers | PyMuPDF, Tesseract OCR, python-docx, Vision API |
| Web UI | Jinja2 + Tailwind CSS + HTMX |
| Шифрование | AES-256-GCM, Argon2id, bcrypt |
| Storage | Local / S3 / Google Drive |

## Быстрый старт

```bash
# Docker Compose
cp .env.example .env    # Настройте API-ключи
docker compose up -d
# → http://localhost:8000

# Или вручную
ssh -L 6333:localhost:6333 -N ugreen &  # туннель к Qdrant
source .venv/bin/activate
make dev
```

### Необходимые ключи

| Ключ | Для чего |
|------|----------|
| `TELEGRAM_BOT_TOKEN` | Telegram бот |
| `TELEGRAM__OWNER_ID` | Ваш Telegram ID |
| `GOOGLE_API_KEY` | Gemini Embedding (бесплатно) |
| `ANTHROPIC_API_KEY` или `OPENAI_API_KEY` | LLM-провайдер |

Настройка моделей, хранилища и шифрования — через веб-панель Settings.
