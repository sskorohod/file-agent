# Шифрование данных

## Главное: ключ — только у вас в голове

Ключ шифрования **нигде не хранится** — ни на сервере, ни в файлах, ни в базе данных.

Вы задаёте **мастер-пароль**. Из него вычисляется ключ шифрования алгоритмом Argon2id (каждая попытка требует 64 МБ памяти и 0.5 секунды). Ключ существует только в оперативной памяти работающего сервера. Выключили — ключ исчез.

На диске остаётся только файл `data/encryption.key` с **солью** и **проверочным токеном**. Ни то, ни другое не позволяет получить ключ без вашего пароля.

---

## Как включить

### Способ 1: через веб-панель (рекомендуется)

1. Откройте **Settings** → вкладка **Security**
2. В секции Data Encryption нажмите **Enable Encryption**
3. Придумайте мастер-пароль (минимум 8 символов)
4. Скачайте **Recovery Key** и сохраните в менеджере паролей
5. Удалите файл recovery key с сервера

После перезапуска сервера зайдите в Settings → Security и введите мастер-пароль для разблокировки. Или через Telegram: `/unlock <пароль> <PIN>`.

### Способ 2: через переменные окружения

```bash
# В .env
ENCRYPTION__FILES=true        # Шифровать файлы на диске
ENCRYPTION__DATABASE=true     # Шифровать текст документов в БД
ENCRYPTION__QDRANT_STRIP=true # Удалить текст из поискового индекса
```

При запуске `make dev` система попросит придумать мастер-пароль (один раз) и будет спрашивать его при каждом запуске. Для автоматического запуска (Docker): `MASTER_PASSWORD=... make dev`.

## Двухфакторная защита

Для максимальной безопасности можно добавить **файл-ключ** — физический второй фактор:

```bash
# Генерация файл-ключа (на USB-флешку)
cd ~/fag && source .venv/bin/activate
python -c "from app.utils.crypto import generate_key_file; generate_key_file('/Volumes/USB/keyfile.secret')"
```

```bash
# В .env — указать путь к файл-ключу
KEY_FILE=/Volumes/USB/keyfile.secret
```

Теперь для разблокировки нужны **оба фактора**: пароль + вставленная флешка. Вытащили флешку — сервер не стартует, данные заблокированы.

## Что шифруется

| Компонент | Что защищено | Флаг |
|-----------|-------------|------|
| **Файлы на диске** | Все документы (PDF, фото, DOCX...) | `ENCRYPTION__FILES` |
| **База данных** | Текст документов, заметки, история чата, summary | `ENCRYPTION__DATABASE` |
| **Поисковый индекс** | Текст удалён из payload (остаются только векторы) | `ENCRYPTION__QDRANT_STRIP` |

Не шифруются: имена файлов, категории, теги, даты — они нужны для навигации. Семантический поиск работает полностью (через векторы).

## Миграция существующих данных

Если у вас уже есть файлы, нужно зашифровать их:

```bash
MASTER_PASSWORD=<пароль> python scripts/migrate_encryption.py --all --verify
```

Флаги:
- `--files` — зашифровать файлы на диске
- `--database` — зашифровать колонки в SQLite
- `--qdrant` — удалить текст из Qdrant
- `--verify` — проверить каждый файл после шифрования
- `--all` — всё сразу

Миграция **безопасна**: уже зашифрованные файлы пропускаются (определяются по magic header `FAGE`), создаётся backup базы данных.

```
                   ┌──────────────┐
                   │  ВАШ ПАРОЛЬ  │
                   └──────┬───────┘
                          │
                          ▼
             ┌────────────────────────┐
             │  Argon2id              │
             │  64 МБ памяти          │ ← соль из data/encryption.key
             │  3 итерации            │
             │         ↓              │
             │  256-бит ключ (в RAM)  │ ← НЕ записывается на диск
             └────────────┬───────────┘
                          │
            ┌─────────────┼──────────────┐
            ▼             ▼              ▼
     ┌─────────┐   ┌──────────┐   ┌──────────┐
     │ Файлы   │   │   БД     │   │  Qdrant  │
     │ AES-256 │   │ AES-256  │   │ text     │
     │  -GCM   │   │  -GCM    │   │ удалён   │
     └─────────┘   └──────────┘   └──────────┘
                          │
                          ▼
                 Выключение сервера
                 → ключ исчез из RAM
```

## Recovery Key

При первой настройке создаётся файл `data/RECOVERY_KEY.txt`. Это **единственный способ** восстановить данные если вы забудете пароль.

**Сохраните его** в менеджере паролей или на бумаге в сейфе. **Удалите с сервера.** Система будет предупреждать, пока файл на диске.

## FAQ

**Как включить/выключить шифрование?**
Включить: Settings → Security → Enable Encryption. Или через .env. Выключить: убрать флаги и не вводить пароль — сервер работает без шифрования, но зашифрованные файлы останутся нечитаемыми до ввода пароля.

**Что видит тот, кто получил доступ к серверу?**
Зашифрованные файлы (бессмысленные байты), зашифрованные колонки в БД (base64 мусор), и файл `encryption.key` с солью — всё это бесполезно без пароля.

**Сколько времени нужно чтобы подобрать пароль?**
Argon2id: 2 попытки/сек на CPU, GPU бесполезен (требуется 64 МБ памяти на попытку). Пароль из 12 символов (буквы+цифры) = ~10^21 вариантов = **16 миллиардов лет** при 2 попытках/сек.

**Я забыл пароль — что делать?**
Используйте recovery key. Если его тоже нет — данные невосстановимы. Это не баг, а фича: никто, включая нас, не может получить доступ к вашим данным.

**Влияет ли на скорость?**
Разблокировка при запуске: ~0.5 сек. Шифрование/расшифровка файлов: +5-10мс (AES аппаратно ускорен на всех современных процессорах). Поиск: без замедления.
