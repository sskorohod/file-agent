# FileAgent — Руководство пользователя

## Что это

FileAgent — персональный AI-агент, который принимает документы и голосовые заметки, автоматически анализирует их, извлекает полезные данные и делает всё доступным через поиск. Работает через Telegram, веб-панель, REST API и MCP (Claude Desktop/Code).

---

## Быстрый старт

### 1. Telegram бот

Найдите вашего бота в Telegram и отправьте `/start`. Вот что он умеет:

**Загрузка файлов** — просто отправьте любой документ (PDF, фото, DOCX). Бот автоматически:
- распознает текст (OCR для фото)
- определит тип документа (паспорт, анализ, счёт)
- извлечёт ключевые данные (даты, суммы, имена)
- сохранит с тегами и категорией
- создаст напоминание, если есть срок действия

**Голосовые сообщения** — отправьте голосовое, бот транскрибирует текст и предложит:
- `🔍 Поиск` — найти информацию в ваших документах
- `📝 Заметка` — сохранить как умную заметку

**Текстовые сообщения** — отправьте текст, бот предложит то же самое: поиск или заметка.

### 2. Веб-панель

Откройте `http://ваш-сервер:8000` в браузере. Логин и пароль задаются в настройках.

### 3. Настройка

Минимум для старта — 3 ключа в файле `.env`:

```
TELEGRAM_BOT_TOKEN=...        # от @BotFather
TELEGRAM__OWNER_ID=...        # ваш Telegram ID (@userinfobot)
GOOGLE_API_KEY=...            # для Gemini Embedding (бесплатно)
```

Плюс один LLM-провайдер: `ANTHROPIC_API_KEY` или `OPENAI_API_KEY`.

## Telegram-команды

### Документы

| Команда | Что делает |
|---------|-----------|
| `/files [категория]` | Список файлов с пагинацией |
| `/recent [N]` | Последние N загрузок (по умолчанию 10) |
| `/search <запрос>` | Семантический поиск по всем документам |
| `/analytics <вопрос>` | Анализ данных с графиками ("проанализируй гемоглобин за год") |
| `/insights` | AI-обзор и рекомендации по категориям |
| `/stats` | Статистика: файлы, размер, расходы LLM |

### Сканирование

| Команда | Что делает |
|---------|-----------|
| `/scan [имя]` | Начать сканирование — отправляйте фото страниц по одной |
| `/done` | Завершить — превью страниц с подтверждением |
| `/cancel` | Отменить сканирование |

Или просто отправьте несколько фото за раз — бот соберёт их в PDF автоматически.

### Умные заметки

| Команда | Что делает |
|---------|-----------|
| `/note <текст>` | Быстрая заметка (сохраняется мгновенно, AI обработает в фоне) |
| `/notes` | Обзор заметок за сегодня по категориям |
| `/today` | Метрики дня: калории, сон, настроение, энергия, вес, стрик |
| `/missing` | Что не заполнено за сегодня (чеклист ✅/❌) |
| `/analyze` | Недельный AI-анализ: паттерны, тренды, рекомендации |

### Шифрование

| Команда | Что делает |
|---------|-----------|
| `/unlock <пароль> [PIN]` | Разблокировать зашифрованные данные |
| `/skills` | Список активных скиллов классификации |

## Умные заметки — как это работает

### Принцип: сохранить мгновенно, понять потом

Когда вы отправляете заметку, она **сохраняется в базу за доли секунды**. AI-обработка происходит в фоне и не мешает сохранению.

### Путь заметки

```
Вы говорите/пишете
    ↓
📝 "Сохранено (#42)" — мгновенно
    ↓ (5-15 секунд, в фоне)
AI определяет:
  • категорию (food, health, business...)
  • настроение, энергию, тональность
  • извлекает людей, суммы, задачи
  • находит связанные заметки
    ↓
📝 Второе сообщение:
  💼 business/meetings
  🏷 клиент, samsung, ремонт
  👤 Евгений (техник)
  ✅ Заказать запчасть до пятницы
  [OK] [Edit] [Archive]
```

### Статусы заметок

| Статус | Значение |
|--------|---------|
| `captured` | Сохранена, ждёт обработки |
| `processing` | AI анализирует |
| `enriched` | Полностью обработана |
| `needs_review` | Обработана, но AI не уверен — проверьте вручную |
| `failed` | Ошибка обработки (заметка сохранена, текст не потерян) |
| `archived` | Архив |

### Категории (12)

| Категория | Что сюда попадает | Пример |
|-----------|------------------|--------|
| food | Еда, напитки, вода | "Съел борщ и два куска хлеба" |
| health | Симптомы, лекарства, врачи | "Болит голова, выпил ибупрофен" |
| fitness | Вес, тренировки, сон | "Спал 7 часов, вес 82 кг" |
| business | Встречи, звонки, задачи | "Звонил клиент по ремонту Samsung" |
| personal | Настроение, размышления | "Сегодня хороший день, настроение 8/10" |
| finance | Расходы, доходы | "Потратил $150 на запчасти" |
| learning | Книги, курсы, инсайты | "Посмотрел видео про диагностику" |
| goals | Планы, цели | "Хочу начать бегать 3 раза в неделю" |
| people | Заметки о людях | "Встретил Ивана, он тоже занимается ремонтом" |
| ideas | Идеи, гипотезы | "А что если сделать приложение для запчастей" |
| family | Семья, события | "День рождения мамы через две недели" |
| auto | Машины, техника, ремонт | "Tesla показывает ошибку подвески" |

### Что AI извлекает из заметок

| Тип данных | Что извлекается | Для чего |
|-----------|----------------|----------|
| **Метрики** | калории, вес, сон, вода, шаги | Графики трендов, недельные отчёты |
| **Сигналы** | настроение (1-10), энергия, sentiment | Корреляции, паттерны |
| **Сущности** | люди, компании, места, проекты | Связи между заметками |
| **Задачи** | действия с приоритетом и дедлайном | Чеклисты, напоминания |
| **Факты** | события, идеи, ключевые инсайты | Обзоры и аналитика |

## Вечерний check-in

В 21:00 (настраивается) бот проверяет, какие данные за день не собраны, и задаёт вопросы:

| Если нет данных о... | Бот спрашивает |
|----------------------|---------------|
| Еда | "🍽 Что ты ел сегодня?" |
| Настроение | "💭 Как настроение? Оцени от 1 до 10" (кнопки) |
| Вес | "💪 Вес сегодня? (в кг)" |
| Сон | "😴 Сколько часов спал прошлой ночью?" |
| Благодарность | "🙏 Что хорошего было сегодня?" |
| Работа | "💼 Итоги дня по работе?" |
| Здоровье | "🏥 Как самочувствие? Есть жалобы?" |

Каждый ответ сохраняется как заметка и обрабатывается AI. В конце — итоги дня с метриками.

Можно пропустить вопрос (кнопка "Пропустить") или завершить раньше ("Завершить").

## Ежедневная и недельная аналитика

### Ежедневная сводка (23:55)

Перед полуночью бот отправляет итоги:
- Количество заметок по категориям
- Калории, настроение, энергия, вес, сон
- Стрик (сколько дней подряд с заметками)

### Недельный отчёт (воскресенье, 22:00)

AI анализирует 7 дней и находит:
- **Паттерны**: "настроение падает по средам"
- **Корреляции**: "спишь лучше в дни с тренировками"
- **Тренды**: "калории на 15% ниже прошлой недели"
- **Алерты**: "пропустил обед 3 дня"
- **Рекомендации**: конкретные, на основе данных

Можно запросить вручную: `/analyze`.

## Веб-панель

### Inbox (Заметки) — `/notes`

Главная страница заметок:
- Все заметки, новые сверху
- Цветные чипы статуса (captured/processing/enriched/needs_review/failed)
- Фильтр по категории и источнику
- Поиск по тексту
- Графики трендов (настроение, калории, вес)
- Корреляции между метриками

### Детали заметки — `/notes/{id}`

Полная информация:
- **Raw text** — оригинальный текст (всегда доступен)
- **AI summary** — краткое описание от AI
- **Категория и теги**
- **Сущности** — люди, компании, места
- **Факты** — извлечённые метрики и события
- **Задачи** — с чекбоксами done/open и приоритетом
- **Связанные заметки** — ссылки с процентом релевантности

### Review Queue — `/notes/review`

Заметки, требующие внимания:
- `needs_review` — AI не уверен в классификации
- `failed` — ошибка обработки

Действия: одобрить, переобработать, архивировать.

### Файлы — `/files`

Все загруженные документы с поиском, фильтрами и пагинацией. Каждый файл можно:
- Просмотреть (PDF inline, изображения)
- Скачать
- Переклассифицировать
- Удалить

### Поиск — `/search`

Семантический поиск по всем документам и заметкам. AI синтезирует ответ из найденных документов.

### Аналитика — `/analytics`

Задайте вопрос — AI найдёт данные в документах и построит график.

### Настройки — `/settings/*`

| Вкладка | Что настраивается |
|---------|------------------|
| Keys | API-ключи провайдеров + внутренние токены |
| LLM | Модели по ролям (classify, search, analyze, extract) |
| Storage | Бэкенд хранения (local, S3, Google Drive) |
| Security | Шифрование: установка, разблокировка, recovery key |
| Advanced | Telegram режим, embedding, системные промпты |

## Шифрование

### Включение

1. Settings → Security → "Set Up Encryption"
2. Придумайте мастер-пароль (рекомендуется 12+ символов)
3. Сохраните Recovery Key в менеджере паролей
4. Удалите файл `data/RECOVERY_KEY.txt` с сервера

### Что шифруется

| Данные | Метод |
|--------|-------|
| Файлы на диске | AES-256-GCM |
| Текст в БД | AES-256-GCM (per-field) |
| Заметки (vault .md) | AES-256-GCM |
| Поисковый индекс | Текст удалён, только векторы |
| API-ключи в БД | Fernet encryption |

### Разблокировка

При каждом перезапуске данные заблокированы. Два способа:
- **Веб**: Settings → Security → ввести пароль
- **Telegram**: `/unlock <пароль> <PIN>`

Ключ хранится только в памяти сервера. При перезагрузке через `--reload` ключ сохраняется в зашифрованном session-файле и восстанавливается автоматически.

## REST API

Для программного доступа создайте API-ключ в Settings → Keys.

```bash
# Поиск
curl -H "Authorization: Bearer ВАШ_КЛЮЧ" \
  "http://localhost:8000/api/v1/search?q=паспорт"

# Список файлов
curl -H "Authorization: Bearer ВАШ_КЛЮЧ" \
  "http://localhost:8000/api/v1/files?limit=10"

# Загрузка
curl -X POST -H "Authorization: Bearer ВАШ_КЛЮЧ" \
  -d '{"filename":"doc.pdf","data":"base64..."}' \
  "http://localhost:8000/api/v1/files/upload"
```

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/v1/stats` | Статистика архива |
| GET | `/api/v1/files` | Список файлов |
| GET | `/api/v1/files/{id}` | Метаданные файла |
| GET | `/api/v1/files/{id}/text` | Извлечённый текст |
| GET | `/api/v1/files/{id}/download` | Скачать файл |
| GET | `/api/v1/search?q=...` | Семантический поиск |
| POST | `/api/v1/files/upload` | Загрузить файл (base64) |
| DELETE | `/api/v1/files/{id}` | Удалить файл |

## Система скиллов

Скиллы — YAML-файлы в папке `skills/`, которые определяют как классифицировать каждый тип документа.

### Встроенные скиллы

- **health** — медицинские документы (анализы, рецепты, страховки)
- **business** — деловые документы (счета, договоры, акты)
- **receipts** — чеки и квитанции
- **personal** — личные документы (паспорта, удостоверения)

### Создание своего скилла

Создайте файл `skills/my_skill.yaml`:

```yaml
name: my_skill
display_name: "Мой скилл"
category: my_category
priority: 5
enabled: true

routing_rules:
  keywords: [ключевое, слово]
  patterns: ["\\d{3}-\\d{7}"]

extraction:
  fields:
    - name: field_name
      description: "Описание поля"
  custom_prompt: "Извлеки данные из документа..."

response_template: |
  📄 {document_type}
  📝 {summary}
```

Скиллы перезагружаются каждые 30 секунд — изменения подхватываются без перезапуска. Также можно редактировать через веб: `/skills/{name}`.

## Inbox Watcher

Система автоматически мониторит папку `~/ai-agent-files/notes/_inbox/` каждые 10 секунд. Если туда попадёт `.md` файл:

1. Содержимое читается
2. Создаётся заметка через CaptureService
3. Запускается async enrichment
4. Оригинал перемещается в `_archive/`

Это позволяет интегрировать любые внешние инструменты — достаточно положить `.md` файл в inbox.

## Obsidian Vault

Обработанные заметки экспортируются как Obsidian-совместимые .md файлы:

```
~/ai-agent-files/notes/
  _inbox/                    ← входящие (мониторятся)
  _archive/                  ← архив оригиналов
  daily/2026-03-28.md        ← ежедневный обзор
  daily/week-2026-W13.md     ← недельный отчёт
  food/meals/...
  health/symptoms/...
  business/meetings/...
  people/clients/...
```

Каждый файл содержит YAML frontmatter с метаданными и wikilinks к связанным заметкам. Vault — это проекция из БД, не основное хранилище. Источник истины — SQLite.

## Советы

- **Говорите естественно** — AI понимает русский, английский и смешанный текст
- **Не бойтесь быть хаотичным** — система сама разложит по категориям
- **Отвечайте на check-in** — чем больше данных, тем точнее аналитика
- **Проверяйте Review Queue** — иногда AI ошибается, ваш фидбек улучшает систему
- **Используйте `/today`** — быстрый способ увидеть свой день в цифрах
- **Храните Recovery Key** — без него зашифрованные данные не восстановить
