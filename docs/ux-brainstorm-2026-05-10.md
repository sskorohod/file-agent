# Brainstorm: как улучшить FAG для пользователя

_Дата: 2026-05-10. Сессия после серии починок (расшифровка, /notes,
intent-routing, edit/delete, LLM-wiki vault, memory audit)._

## Кто пользователь и как он реально работает

Из истории сессий и архива видно три **главных рабочих сценария**:

1. **Дневной поток заметок (heavy use, ~10/день в активные дни)** —
   надиктовать голосом или написать текстом «Check-in / Энергия 7»,
   «На завтрак был бутерброд», «Договорился с Ваней на 10:30»,
   «Идея для CRM: блокировать слот техника». Потом — **аналитика**:
   как менялось настроение неделю назад? что я ел в среду? какие
   идеи накопились по работе?

2. **Документы (medium use, ~50 файлов)** — паспорт, права, SSN,
   медицинские выписки, налоговые формы, pay-stubs, USCIS-уведомления.
   Загрузил один раз. Найти **через несколько месяцев** когда
   «приходи в DMV с правами и USCIS-формой».

3. **Связи (rarely used today, потенциально killer-feature)** —
   «У Inha готовы все документы для DMV?», «Какие pay-stubs я
   уже отправил в иммиграционное?», «Что я говорил про CRM на
   прошлой неделе и что из этого сделал?».

## Что сейчас работает хорошо

- ✅ Загрузка файла → точная классификация → шифрование sensitive
- ✅ Поиск файлов по названию документа («найди паспорт» → 2 паспорта
  с дисамбигуацией кнопками)
- ✅ Заметки — пагинация, edit/delete, поиск по дню, semantic search
- ✅ Auto-delete sensitive файлов из чата через 15 мин
- ✅ PIN на чувствительные документы
- ✅ Cognee-граф интегрирован (правда сидит на старых данных, нужен
  re-ingest)

## Что **не используется**, хотя могло бы давать ценность

### A. Аналитика заметок — дыра №1

У нас в SQLite уже есть **таблицы** `note_enrichments`,
`mood_score`, `sentiment`, `energy`, `note_facts`, `note_data`,
`habit_entries`, `food_entries`, `water_entries`, `medical_metrics`,
`personal_baselines`, `lag_correlations`, `anomaly_alerts`,
`checkin_signal_stats`, `checkin_state` — **тонна метрик**, которые
никогда не показываются пользователю.

**Что предложить:**

1. **`/dashboard`** — Telegram-команда, отвечает картинкой-графиком
   (matplotlib → PNG → bot.send_photo) с:
   - mood-графиком за последние 30 дней
   - sleep-часы поверх mood (corellation)
   - energy-уровень
   - частота тренировок
   - кофе/еда-патерны
2. **`/today`** — что я делал/говорил/ел сегодня. Один экран, без
   запроса.
3. **`/week`** — недельный отчёт с инсайтами от LLM («ты пил кофе
   после 17:00 — 4 раза, потом плохо спал в эти ночи»).
4. **Inline-charts через Telegraph / Quickchart.io** — без своей
   инфраструктуры, просто bot.send_photo с URL картинки.
5. **Streaks** — «🔥 7 дней подряд checkin записывал». Как Duolingo.
   Подключается к существующему `checkin_state`.
6. **Anomaly alerts** — таблица уже есть. Бот сам пингует «сегодня
   твой energy на 30% ниже среднего за месяц, проверь сон»
   (использовать уже существующий `_daily_advice_loop`).

### B. Голосовой ввод — недоиспользован

Сейчас: голосовое → Whisper → текст → "поиск или заметка?"
(после PR #11 — сразу поиск если transcript начинается с
«найди»).

**Дополнить:**

1. **Голосовые команды для аналитики** — «как я спал на этой
   неделе» / «покажи дашборд» / «что я ел вчера».
2. **`reply` голосом на сообщение бота** — например, бот выдал
   `📝 заметка: ...` → ты в reply шлёшь голос «дополни: ещё
   добавил молока в кофе» → автоматическая правка существующей
   заметки (не новой). Сейчас reply игнорируется.
3. **Голосовой запрос «что было вчера в 3 часа» с грубым тайм-биннингом**
   — заметки + локации + расписание из календаря (если интегрируем).

### C. Telegram-фичи которые игнорируются

| Фича Telegram | Что даёт | Стоимость |
|---|---|---|
| **Inline mode** (`@vskorohodbot passport`) | поиск из любого чата без открытия диалога с ботом | средне (нужен webhook + `answer_inline_query`) |
| **Bot menu** (`set_chat_menu_button`) | синяя кнопка «Меню» с быстрыми действиями вместо `/команд` | дёшево |
| **Bot commands** (`set_my_commands` per scope) | автокомплит `/today`, `/notes`, `/files`, `/dashboard` в нативном UI | дёшево |
| **Topics в группе** | если ты заведёшь группу-архив, каждая категория документов = отдельный topic, у каждого пинн с актуальным резюме | средне |
| **Polls для habit-tracking** | «Тренировался сегодня? 👍 / 👎» — ответ → запись в `habit_entries` | дёшево |
| **Quick reactions** на сообщения бота | 👍/👎/🤔 на реплай бота → feedback signal для ranking поиска | средне |
| **Live location** | «где я оставил паспорт» — бот пишет в заметку при загрузке файла локацию (если `bot.send_location` поддержан) | средне, но privacy-важно |
| **Animated stickers как ack** | «✅ записано» с маленькой анимацией Telegram-стикера вместо текста | дёшево, очень телеграмно |
| **Reply keyboard вместо inline** для частых действий | физическая клавиатура внизу чата с «📝 заметка / 📊 dashboard / 🔍 искать» | дёшево |
| **Mini-app** (Web App via `web_app_url`) | внутри Telegram открывается полная панель с графиками и таблицей файлов | дорого, но один раз |
| **Scheduled messages** | «напомни мне через час про DMV» — нативный TG mechanism, не наш cron | средне |
| **Bot payments** | оплата USCIS-fee через бота (Telegram Stars или провайдер) | дорого |
| **Channel mode** | публичный канал «sskorohod's life log» закрытый только тебе — безопасный архив | дёшево |

## Что технически уже **готово** в коде, но не подключено к UX

Скан кода и schema показал зомби-функционал:

- **`reminders` table + `_reminder_loop`** в main.py — `bot.send_message`
  по расписанию. Только используется как deadline-нотификация по
  expiry_date файлов. Но schema поддерживает кастомные напоминания —
  не доступно через `/remind <text> in 2h`.
- **`anomaly_alerts` + `lag_correlations` + `personal_baselines`** —
  готовая инфра для «ты сегодня странно споришь, проверь, не на
  голодный ли желудок». Никогда не дёргается.
- **`note_relations` table (1487 рядов!)** — заметки уже как-то
  связаны между собой, но нигде не показывается. «Похожие заметки»
  снизу транскрипта — мгновенный win.
- **`magic_link_tokens`** — одноразовые ссылки на web. Не используется.
  Можно делать «открой dashboard в браузере» из бота через `bot.send_message`
  с https://fag.../dashboard?t=onetime.
- **`api_keys` + `entity_aliases`** — таблицы есть, наполнение есть, но
  поверхность через бота — нет.

## Что менять в **существующей** функциональности

### Заметки

- **Сейчас** в /notes показывается титул + дата. **Идея:** добавить
  цветной индикатор по `mood_score` (🟢/🟡/🔴) + emoji по
  `category` (🍎 еда / 🛏 сон / 🏋 фитнес / 💼 работа / 💡 идея).
- **Сейчас** edit заменяет полный текст. **Идея:** через `reply` к
  сообщению бота с заметкой можно «дописать» (LLM сам мерджит).

### Файлы

- **Сейчас** при поиске нужно знать тип («найди паспорт»).
  **Идея:** «найди что мне нужно для DMV» → bot собирает из
  graph: DL + SSN + paystubs + USCIS-approval (multi-doc bundle).
- **Сейчас** sensitive открываются по PIN. **Идея:** разные TTL для
  разных категорий (15 мин для passport, 5 мин для SSN).
- **Сейчас** реклассификация — manual ручка. **Идея:** автоматический
  re-classify когда меняется skill YAML или promp.

### Поиск

- **Сейчас** intent-aware: notes vs files. **Идея:** добавить
  третий режим — **«граф»** (cognee.recall) для запросов типа «всё
  что связано с Inha».
- **Сейчас** ответ — просто text + buttons. **Идея:** для
  multi-doc-bundle ответа возвращать **карточкой** (HTML с
  миниатюрами через Telegraph API).

### Dashboard / Web UI

- **Сейчас** есть `/settings`, `/files`, `/skills`, `/analytics`.
  **Идея:** **/timeline** — день за днём, что произошло (загружено
  файлов, наговорено заметок, мудстрики).
- **Mini-app** в Telegram — то же самое внутри клиента, без
  переключения в браузер.

## Что **не делать** (anti-pattern для FAG)

- **Не делать чат-бота-помощника-всезнайку.** ChatGPT уже есть.
  Ценность FAG — это **твой архив + твои метрики**, а не LLM-talker.
- **Не строить web SPA.** HTMX-форма и бот — достаточно. Mini-app
  только если есть конкретная фича, которая не лезет в чат.
- **Не делать calendar/email-интеграцию широкого профиля.** Это
  бесконечный rabbit hole. Если нужен календарь — pull-only через
  существующий gcal MCP, без двусторонней синхронизации.
- **Не делать публичные API.** Это персональный агент. Каждый новый
  surface = новый attack-vector.

## Связь с memory audit

Brainstorm выше предполагает что **memory layer работает** —
запросы возвращают правильные документы и заметки без галлюцинаций.
Если пробелы из `docs/memory-audit-2026-05-10.md` (outbox,
reconciliation, cognee mandatory, dedup) не закрыты, **большинство
предложенных фич будут давать ненадёжные ответы**.

Сначала Sprint D + E (фундамент памяти), потом 1-3 топ-предложения
из этого документа. Sprint J (edit/delete заметок) уже сделан и
закрывает один пункт.
