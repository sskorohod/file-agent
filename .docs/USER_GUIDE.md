# FileAgent — Руководство пользователя

## Быстрый старт

### 1. Запуск сервера

```bash
# SSH-туннель к Qdrant (требуется перед стартом)
ssh -L 6333:localhost:6333 -L 6334:localhost:6334 -N ugreen &

# Запуск
source .venv/bin/activate
make dev
```

Сервер доступен на `http://localhost:8000`.

### 2. Первый вход в веб-дашборд

Откройте `http://localhost:8000` → появится страница входа.
Введите email и пароль (настраиваются в `.env`).

## Работа с Telegram ботом

### Загрузка файлов

**Одиночный файл:**
Просто отправьте файл (PDF, фото, DOCX) боту. Он автоматически:
1. Распознает текст
2. Определит категорию и тип документа
3. Извлечёт ключевые данные
4. Даст подробное описание и рекомендации
5. Создаст напоминание если есть срок действия
6. Сохранит в базу знаний

**Многостраничный документ (пошагово):**
```
/scan Паспорт жены     ← начать сканирование
[фото страницы 1]      ← бот: "Страница 1 добавлена"
[фото страницы 2]      ← бот: "Страница 2 добавлена"
/done                   ← бот показывает превью
[✅ Собрать PDF]        ← подтвердить → обработка
```

**Многостраничный документ (одним сообщением):**
Выберите несколько фото в галерее и отправьте одним сообщением.
Бот автоматически соберёт их в PDF и обработает.

**Отмена сканирования:**
```
/cancel               ← отменить текущую сессию
```

### Поиск по документам

Просто напишите вопрос текстом:
```
Когда истекает мой паспорт?
Какие лекарства мне прописали?
Покажи все документы по u4u
```

Или используйте команду:
```
/search водительские права
```

### Голосовые сообщения

Отправьте голосовое сообщение — бот распознает речь и предложит:
- 🔍 **Поиск** — искать по документам
- 📝 **Заметка** — сохранить как заметку с тегами

### Ежедневные советы

Два раза в день бот присылает мотивирующий совет:
- 🌅 **9:00** — что важного сделать сегодня
- 🌙 **20:00** — обзор дня и план на завтра

## Работа с веб-дашбордом

### Dashboard (/)

Главная страница с обзором:
- **6 KPI карточек** — файлы, векторы, запросы, LLM вызовы, напоминания, ошибки
- **Activity** — лента последних действий (загрузки + поиски)
- **Categories** — распределение документов по категориям
- **AI Usage** — статистика использования LLM
- **Reminders** — ближайшие напоминания
- **System** — статус Qdrant, хранилища, pipeline

### Files (/files)

Список всех файлов:
- **Фильтры** — поиск по имени, фильтр по категории
- **Мобильный вид** — карточки вместо таблицы
- **Действия** — скачать, переклассифицировать, удалить

### File Detail (/files/{id})

Подробности файла:
- **Слева** — summary, extracted fields, metadata, теги
- **Справа** — превью (PDF inline, изображения с zoom)
- **Zoom** — Fit / 100% / + / - / колёсико мыши
- **Processing Log** — шаги обработки с таймингами

### Search (/search)

Семантический поиск:
- Введите вопрос → AI даст ответ на основе документов
- Результаты с score-барами и preview текста

### Analytics (/analytics)

Аналитика данных:
- Введите вопрос → график + текстовый анализ
- Карточки показателей сгруппированы по метрике
- Сырая таблица доступна по кнопке

### Insights (/insights)

AI обзор по категориям:
- Обзор ситуации, ключевые проблемы, рекомендации
- Кнопка "Обновить" для пересчёта

### Skills (/skills)

Управление навыками классификации:
- YAML-редактор для каждого навыка
- Горячая перезагрузка (автоматически каждые 30 сек)

### Settings (/settings)

Конфигурация:
- **API Keys** — создание/удаление ключей (lite/full)
- **LLM Models** — выбор провайдера и модели для каждой роли
- **System Prompts** — настройка промптов поиска и классификации
- **MCP подключение** — инструкция для Claude Code / Claude Desktop

### Другие страницы

- **Folders** (/folders) — организация файлов в папки
- **Reminders** (/reminders) — все напоминания с документами
- **Duplicates** (/duplicates) — поиск и разрешение дубликатов
- **Logs** (/logs) — история обработки файлов

## REST API

### Авторизация

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" http://localhost:8000/api/v1/stats
```

Ключи создаются в Settings. Два режима:
- **lite** — только поиск (векторные чанки)
- **full** — поиск + LLM синтез ответа

### Эндпоинты

```
GET  /api/v1/stats                    — статистика
GET  /api/v1/files?category=&limit=   — список файлов
GET  /api/v1/files/{id}               — метаданные файла
GET  /api/v1/files/{id}/text          — извлечённый текст
GET  /api/v1/files/{id}/download      — скачать файл
GET  /api/v1/search?q=&top_k=         — семантический поиск
POST /api/v1/files/upload             — загрузить файл (base64)
DELETE /api/v1/files/{id}             — удалить файл
```

### MCP (Claude Code / Claude Desktop)

```json
{
  "mcpServers": {
    "file-agent": {
      "url": "http://localhost:8000/mcp/sse",
      "headers": {"Authorization": "Bearer YOUR_KEY"}
    }
  }
}
```

## Типичные сценарии

### Загрузка паспорта
1. Сфотографируйте страницы паспорта
2. Отправьте через `/scan Загранпаспорт` или media group
3. Получите: тип, ФИО, даты, рекомендации по продлению
4. Автоматическое напоминание за 6 месяцев до истечения

### Анализ крови
1. Сфотографируйте результат анализа
2. Отправьте в бот
3. Получите: расшифровку показателей, отклонения от нормы, рекомендации
4. Позже: `/analytics проанализируй гемоглобин` → график трендов

### Бизнес-документ
1. Отправьте PDF контракта или счёта
2. Получите: сумму, сроки, обязательства, план действий
3. Автоматическое напоминание если есть дедлайн
